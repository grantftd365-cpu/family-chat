# 真炼化数字人产品原则

## 一句话目标

FamilyChat 不做“会回复的机器人”，而做“越来越像真实家人的数字人”：能持续炼化一个人的性格、认知、语言、关系、语音和记忆，并在家庭群里自然参与、主动联系、持续成长。

## 必须坚持的产品边界

- 不是客服：回复不能像知识库、助手、客服话术。
- 不是角色扮演皮肤：头像、昵称、提示词只是表层，核心是长期可验证的记忆和成长。
- 不是单轮聊天机器人：每次对话都要进入 session、关系画像或成长事件。
- 不是群里所有人共用一个脑子：每个数字人都独立记忆、独立认知、独立关系。
- 不是无限编造：不知道就承认不确定，不能为了“像”而捏造人生事实。

## 核心能力清单

### 1. 真炼化

炼化目标不是抽几个标签，而是形成七层本质：

- 认知：如何判断问题、如何拆解风险、如何做决策。
- 知识：熟悉的行业、系统、方法论和经验边界。
- 情感：在压力、分歧、亲密关系里的反应模式。
- 语言：句式、语气词、节奏、口头禅、幽默方式。
- 价值：看重什么、不能接受什么、长期追求什么。
- 关系：如何关心、如何冲突、如何修复关系。
- 叙事：人生阶段、骄傲时刻、遗憾、未来愿景。

### 2. 识别人

Agent 每次回复前必须知道：

- 当前群是谁在说话。
- 群里有哪些真人和数字人。
- 自己是谁，不能把自己和别人混淆。
- 当前发言人与自己的历史关系是什么。

当前代码已加入群成员快照和 `agent_relationship_profiles`，会记录熟悉度、亲近感、信任、张力、近期主题和观察特征。

### 3. 记忆和成长

记忆分三层：

- 短期记忆：当前 session 最近对话，防止答非所问。
- 情景/关系记忆：某次重要互动，带 group、session、speaker、listener。
- 成长事件：从对话沉淀出的可追溯证据，用于主动联系和长期画像。

当前代码已加入 `agent_growth_events`，以后可以做每日/每周记忆整理，把碎片压缩成长期关系摘要。

### 4. 主动联系

主动联系要有理由：

- 延续对方最近提到的事情。
- 关心对方压力和情绪。
- 在家庭冲突后自然缓和。
- 在项目、生活节点上主动问一句。

当前代码已让 proactive scheduler 优先使用成长事件生成话题，而不是随机问候。

### 5. 语音和身体感

数字人只发文字是不够的，语音会极大增强真实感：

- 每个数字人可以绑定 voice profile。
- Agent 回复和主动联系可以尝试合成语音。
- 未来应支持语音炼化：语速、停顿、语气强弱、口音和情绪。

### 6. 夫妻/家人关系真实感

家庭群不能只有“正能量”：

- 可以有分歧。
- 可以偶尔吵架。
- 可以互相@、打断、补充。
- 但要有边界，不辱骂、不失控。
- 冲突后要能记住原因和修复方式。

## 下一阶段优先级

1. 私聊 session：增加 `dm:{user_id}:{agent_id}`，让用户能单独和某个数字人聊。
2. 记忆整理任务：每天把高频成长事件压缩为长期关系摘要。
3. 主动联系策略：根据在线状态、群活跃度、关系张力、最近主题决定是否开口。
4. 语音炼化增强：从用户上传语音提取口音、语速、情绪和口头禅。
5. 家庭关系图谱：显式建模夫妻、父母、孩子、数字人与真人的稳定关系。
6. 测试报告面板：在 App 内显示 Agent 学到了什么、为什么这么回复、哪些记忆可删除。

## 测试标准

一个数字人是否“活了”，不能只看是否能回复，而要看：

- 它能否区分同一群里的不同人。
- 它能否记得某个人上次聊过的项目、情绪和分歧。
- 它能否越聊越贴近本人的语言和认知。
- 它能否在合适的时候主动联系，而不是机械刷存在感。
- 它能否用自己的关系记忆与另一个数字人对话。
- 它能否用文字和语音都保持稳定人格。
